# 融智通 - 模块2：发票查重验真 详细设计

## 一、模块概述

### 1.1 业务背景
融资租赁项目中，发票是证明设备采购真实性的核心凭证。
常见风险：
- **重复融资**：同一批设备（同一批发票）在多个租赁公司重复申请融资
- **虚假发票**：伪造或篡改发票，骗取融资
- **发票信息不符**：发票金额、设备信息与清单不匹配

目前多数租赁公司靠人工核验发票，效率低且容易遗漏。

### 1.2 核心功能
1. OCR自动识别发票信息
2. 调用税务API验真（国税总局接口）
3. 跨项目/跨客户发票查重（核心亮点）
4. 发票信息与清单自动交叉比对

### 1.3 价值量化
| 指标 | 现状 | 使用后 | 提升 |
|------|------|--------|------|
| 单张发票核验时间 | 5-10分钟 | 10秒 | 50倍 |
| 重复发票发现率 | ~60% | 99%+ | +39% |
| 虚假发票识别率 | 依赖经验 | 95%+ | 大幅提升 |
| 批量处理能力 | 50张/天/人 | 1000+张/天 | 20倍 |

### 4.2 详细步骤

#### 步骤1：上传发票
```
操作：
1. 用户进入"发票查重验真"页面
2. 选择关联项目（必选）
3. 上传发票文件（支持批量）
4. 支持格式：JPG/PNG/PDF
5. 单个文件≤50MB，单次≤100张

上传校验：
- 文件格式检查
- 重复文件检测（MD5去重）
- 文件质量预检（分辨率≥300DPI建议）
```

#### 步骤2：OCR识别
```
处理过程：
1. 调用PaddleOCR/百度OCR增值税发票识别API
2. 提取发票全部字段
3. 格式校验（发票代码位数、金额格式等）
4. 置信度标注

支持发票类型：
- 增值税专用发票
- 增值税普通发票
- 电子专用发票（全电发票）
- 电子普通发票
- 机动车销售统一发票（设备车辆类）
```

#### 步骤3：发票验真
```
验真方式：
1. 国税总局全国增值税发票查验平台API
   - 输入：发票代码、发票号码、开票日期、校验码/金额
   - 输出：发票状态（正常/作废/红冲/失控）

2. 验真结果处理：
   - ✅ 正常：发票有效
   - ❌ 作废：发票已作废
   - ⚠️ 红冲：发票已红字冲销
   - 🚫 失控：税务机关标记为失控发票
   - ❓ 查无：未查到该发票（可能伪造）

3. 频率限制处理：
   - 国税API有调用频率限制
   - 实现请求队列，控制调用速度
   - 批量验真时分批处理
```

#### 步骤4：跨项目查重（核心功能）
```
查重范围：
1. 本公司内部查重
   - 同一项目内重复
   - 不同项目间重复
   - 不同客户间重复

2. 查重策略：
   - 精确匹配：发票代码+发票号码 完全一致
   - 模糊匹配：金额+日期+卖方 组合匹配
   - 关联匹配：同一购买方+相近金额+相近日期

3. 查重结果分级：
   ■ 高风险（红色）：
     完全相同的发票出现在不同项目中
     → 极可能重复融资
   
   ■ 中风险（黄色）：
     相同卖方+相近金额+相近时间段
     → 可能拆分发票或关联交易
   
   ■ 低风险（蓝色）：
     部分信息相似
     → 建议人工复核
   
   ■ 无风险（绿色）：
     未发现任何重复
     → 通过查重
```

#### 步骤5：清单交叉比对
```
比对维度：
1. 金额比对
   - 发票总金额 vs 清单总金额
   - 允许误差范围（可配置，如±1%）

2. 设备比对
   - 发票货物明细 vs 清单设备明细
   - 品名/规格匹配（LLM辅助语义匹配）

3. 卖方比对
   - 发票销售方 vs 清单供应商
   - 企业名称模糊匹配

4. 比对结果：
   ✅ 一致：所有维度匹配
   ⚠️ 部分一致：金额一致但明细有差异
   ❌ 不一致：存在明显不符
```

## 六、查重算法设计

### 6.1 精确查重

```python
def exact_dedup(invoice):
    """精确匹配：发票代码+发票号码"""
    existing = db.query(
        "SELECT * FROM invoices WHERE "
        "invoice_code = ? AND invoice_no = ? "
        "AND project_id != ?",
        invoice.code, invoice.no, invoice.project_id
    )
    if existing:
        return {
            "level": "HIGH",
            "reason": "完全相同的发票出现在其他项目中",
            "matches": existing
        }
    return None
```

### 6.2 模糊查重

```python
def fuzzy_dedup(invoice):
    """模糊匹配：相同卖方+相近金额+相近日期"""
    similar = db.query(
        "SELECT * FROM invoices WHERE "
        "seller_tax_no = ? AND "
        "ABS(total_amount - ?) / ? < 0.05 AND "  # 金额误差5%以内
        "ABS(DATEDIFF(invoice_date, ?)) <= 30 AND "  # 日期30天以内
        "project_id != ?",
        invoice.seller_tax_no,
        invoice.total, invoice.total,
        invoice.date,
        invoice.project_id
    )
    if similar:
        return {
            "level": "MEDIUM",
            "reason": "相同卖方、相近金额和日期",
            "matches": similar
        }
    return None
```

### 6.3 发票指纹

```python
def generate_invoice_fingerprint(invoice):
    """生成发票指纹用于快速查重"""
    # 组合关键字段生成哈希
    raw = f"{invoice.code}|{invoice.no}|{invoice.date}|{invoice.total}"
    fingerprint = hashlib.sha256(raw.encode()).hexdigest()
    
    # 存入Redis用于实时查重
    redis.sadd("invoice_fingerprints", fingerprint)
    
    return fingerprint
```

## 七、与国税API对接

### 7.1 接口说明

```
接口：全国增值税发票查验平台
地址：https://inv-veri.chinatax.gov.cn（示例）

请求参数：
{
    "fpdm": "发票代码",
    "fphm": "发票号码",
    "kprq": "开票日期（YYYYMMDD）",
    "fpje": "开具金额（不含税，专票）/ 校验码后6位（普票）"
}

返回结果：
{
    "code": "0001",  // 查验成功
    "data": {
        "fpdm": "发票代码",
        "fphm": "发票号码",
        "kprq": "开票日期",
        "gfmc": "购买方名称",
        "xfmc": "销售方名称",
        "je": "金额",
        "se": "税额",
        "jshj": "价税合计",
        "zfbz": "作废标志",
        "fpzt": "发票状态"
    }
}
```

### 7.2 调用策略

```
频率控制：
- 单张发票查验间隔≥1秒
- 批量查验使用队列，QPS控制在1-2
- 同一发票24小时内不重复查验（缓存结果）

错误处理：
- 验证码/滑块验证 → 提示人工处理或使用商业API
- 接口超时 → 重试3次，间隔递增
- 接口不可用 → 标记为"待验真"，后续重试

商业API备选：
- 百度AI发票验真
- 航天信息API
- 税友集团API
- 费用约0.1-0.5元/次
```

## 八、异常处理

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| OCR识别失败 | 图片模糊/角度偏 | 提示重新拍照上传 |
| 验真失败 | API异常 | 加入重试队列，标记待验真 |
| 高风险查重 | 同一发票多项目 | 弹窗预警，通知风控人员 |
| 金额不匹配 | 发票合计≠清单 | 标注差异，人工确认 |
