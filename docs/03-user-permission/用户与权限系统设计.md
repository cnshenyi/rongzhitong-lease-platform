# 融智通 - 用户与权限系统设计

## 一、用户体系设计

### 1.1 用户角色定义

#### 超级管理员 (Super Admin)
- **权限范围**：全部功能，无限制
- **职责**：
  - 系统配置与维护
  - 用户管理与权限分配
  - 数据备份与恢复
  - 系统监控与日志审计
- **数量限制**：1-2人
- **典型场景**：IT管理员、系统运维人员

#### 管理员 (Admin)
- **权限范围**：业务功能 + 部分管理功能
- **职责**：
  - 组织架构管理
  - 用户权限分配（本组织内）
  - 业务数据监控
  - 模板配置
- **数量限制**：2-5人
- **典型场景**：业务部门负责人、IT支持人员

#### 业务主管 (Manager)
- **权限范围**：查看所有项目、审批、统计分析
- **职责**：
  - 项目审批与监督
  - 团队管理
  - 业绩考核
  - 风险把控
- **数量限制**：5-10人
- **典型场景**：业务经理、风控经理

#### 业务员 (User)
- **权限范围**：自己的项目、核心业务功能
- **职责**：
  - 项目录入与维护
  - 文档处理（清单、发票、报告等）
  - 客户对接
  - 日常业务操作
- **数量限制**：20-100人
- **典型场景**：业务员、客户经理

#### 只读用户 (Viewer)
- **权限范围**：仅查看权限
- **职责**：
  - 数据查询
  - 报表查看
  - 统计分析
- **数量限制**：不限
- **典型场景**：财务人员、审计人员、高层领导

## 二、权限控制模型

### 2.1 RBAC权限模型

```
用户 (User) ──┐
              ├──> 角色 (Role) ──> 权限 (Permission) ──> 资源 (Resource)
组织 (Org) ───┘
```

#### 核心概念

**用户 (User)**
- 系统使用者
- 属于一个或多个组织
- 拥有一个或多个角色

**组织 (Organization)**
- 公司、部门、团队
- 树形结构，支持多级
- 数据隔离单元

**角色 (Role)**
- 权限的集合
- 可自定义
- 支持继承

**权限 (Permission)**
- 对资源的操作许可
- 格式：资源:操作（如 project:create）

**资源 (Resource)**
- 系统功能模块
- 数据对象

### 2.2 权限粒度

#### 功能权限（模块级）
```
- 项目管理
  - 项目列表
  - 项目详情
  - 项目创建
  - 项目编辑
  - 项目删除
  
- AI工具
  - 清单识别
  - 发票查重
  - 尽调报告
  - 合同审核
  - 方案计算
  - 智能问答
  
- 数据统计
  - 业务统计
  - 效率分析
  - 风险报表
  
- 系统设置
  - 用户管理
  - 角色管理
  - 权限管理
  - AI配置
  - 模板管理
```

#### 操作权限（动作级）
```
- View (查看)
- Create (新增)
- Update (编辑)
- Delete (删除)
- Approve (审批)
- Export (导出)
- Import (导入)
```

#### 数据权限（数据级）
```
- 全部数据：所有组织的数据
- 本组织数据：当前组织及子组织的数据
- 本部门数据：当前部门的数据
- 个人数据：仅自己创建的数据
- 自定义：灵活配置
```

## 三、组织架构设计

### 3.1 组织结构

```
公司 (Company)
│
├── 部门A (Department A)
│   ├── 团队A1 (Team A1)
│   │   ├── 用户1
│   │   └── 用户2
│   └── 团队A2 (Team A2)
│       ├── 用户3
│       └── 用户4
│
└── 部门B (Department B)
    ├── 团队B1 (Team B1)
    │   ├── 用户5
    │   └── 用户6
    └── 团队B2 (Team B2)
        ├── 用户7
        └── 用户8
```

### 3.2 组织属性

```
组织 (Organization)
├── id: 唯一标识
├── name: 组织名称
├── type: 组织类型（公司/部门/团队）
├── parent_id: 父组织ID
├── level: 层级（1/2/3...）
├── path: 路径（如：/1/2/5）
├── leader_id: 负责人ID
├── status: 状态（启用/禁用）
├── created_at: 创建时间
└── updated_at: 更新时间
```

### 3.3 数据隔离策略

#### 完全隔离
- 不同公司之间数据完全隔离
- 适用于SaaS多租户模式

#### 层级隔离
- 上级可查看下级数据
- 下级不可查看上级数据
- 同级之间不可互相查看

#### 自定义隔离
- 通过权限配置灵活控制
- 支持跨部门协作

## 六、权限判断逻辑

### 6.1 前端权限控制

#### 路由守卫
```javascript
router.beforeEach((to, from, next) => {
  // 1. 检查是否登录
  if (!isLoggedIn()) {
    return next('/login')
  }
  
  // 2. 检查路由权限
  if (to.meta.permission) {
    if (!hasPermission(to.meta.permission)) {
      return next('/403')
    }
  }
  
  next()
})
```

#### 按钮权限指令
```vue
<template>
  <!-- 有权限才显示 -->
  <el-button v-permission="'project:create'">
    新建项目
  </el-button>
  
  <!-- 有任一权限即显示 -->
  <el-button v-permission="['project:edit', 'project:delete']">
    操作
  </el-button>
</template>
```

### 6.2 后端权限控制

#### 装饰器验证
```python
from functools import wraps
from fastapi import HTTPException

def require_permission(permission: str):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            current_user = get_current_user()
            if not has_permission(current_user, permission):
                raise HTTPException(status_code=403, detail="无权限")
            return await func(*args, **kwargs)
        return wrapper
    return decorator

# 使用
@router.post("/projects")
@require_permission("project:create")
async def create_project(data: ProjectCreate):
    pass
```

#### 数据权限过滤
```python
def get_projects(user: User):
    query = db.query(Project)
    
    # 根据用户数据权限过滤
    if user.data_scope == "ALL":
        # 全部数据
        pass
    elif user.data_scope == "ORG":
        # 本组织数据
        query = query.filter(Project.org_id == user.org_id)
    elif user.data_scope == "DEPT":
        # 本部门数据
        query = query.filter(Project.dept_id == user.dept_id)
    elif user.data_scope == "SELF":
        # 个人数据
        query = query.filter(Project.creator_id == user.id)
    
    return query.all()
```

### 7.1 日志记录范围

#### 用户操作
- 登录/登出
- 密码修改
- 个人信息修改

#### 数据操作
- 创建记录
- 编辑记录
- 删除记录
- 批量操作

#### 权限操作
- 角色创建/修改/删除
- 权限分配
- 用户启用/禁用

#### 敏感操作
- 数据导出
- 系统配置修改
- AI配置修改

### 7.2 日志字段

```
操作日志 (OperationLog)
├── id: 日志ID
├── user_id: 操作人ID
├── user_name: 操作人姓名
├── module: 模块（项目管理/用户管理/...）
├── action: 操作（创建/编辑/删除/...）
├── resource_type: 资源类型（项目/用户/角色/...）
├── resource_id: 资源ID
├── content: 操作内容（JSON）
├── ip: IP地址
├── user_agent: 浏览器信息
├── status: 状态（成功/失败）
├── error_msg: 错误信息
├── created_at: 操作时间
```
