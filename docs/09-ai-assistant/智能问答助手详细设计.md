# 融智通 - 模块6：智能问答助手 详细设计

## 一、模块概述

### 1.1 业务背景
融资租赁行业涉及大量专业知识：
- 法律法规（金融租赁管理办法、民法典、税收政策等）
- 行业规范（融资租赁企业监管评级、风控指引等）
- 公司内部制度（审批流程、风控标准、操作手册等）
- 业务知识（会计处理、税务筹划、残值评估等）

新人培训成本高，老员工也需要频繁查阅资料。

### 1.2 核心功能
基于RAG（检索增强生成）技术的智能问答助手：
1. 构建融资租赁知识库（法规+内部制度+行业知识）
2. 自然语言提问，AI精准回答
3. 引用原文出处，确保可追溯
4. 多轮对话，上下文理解
5. 支持文档上传，自动入库

### 1.3 价值量化
| 指标 | 现状 | 使用后 | 提升 |
|------|------|--------|------|
| 问题解答时间 | 10-30分钟 | 10秒 | 100倍 |
| 知识覆盖率 | 依赖个人 | 全量知识库 | 100% |
| 新人培训周期 | 3-6个月 | 1-2个月 | 60% |
| 专业问答成本 | 专家时间 | AI+验证 | -80% |

### 3.3 文档处理流程

```
上传文档 → 文本提取 → 分段(Chunk) → 向量化 → 存储

详细步骤：
1. 文本提取
   - PDF → pdfplumber/PyPDF2提取文本
   - Word → python-docx提取文本
   - Excel → pandas提取数据
   - 扫描件 → OCR识别

2. 文本分段（Chunking）
   - 按段落/章节分段
   - 每段200-500字
   - 段落间有50字重叠
   - 保留元数据（文档名、章节、页码）

3. 向量化（Embedding）
   - 使用向量模型：
     - 中文：text2vec-large-chinese
     - 通义千问 text-embedding-v2
     - OpenAI text-embedding-3-small
   - 生成768/1536维向量

4. 存储
   - 向量数据库：PostgreSQL + pgvector
   - 或 Milvus / Chroma
   - 存储：向量 + 原文 + 元数据
```

## 四、RAG架构设计

### 4.1 RAG流程

```
用户提问
    │
    ▼
┌──────────┐
│ 问题理解  │ ← LLM改写/分解问题
└────┬─────┘
     │
     ▼
┌──────────┐
│ 向量检索  │ ← 从知识库检索相关文档
└────┬─────┘
     │
     ▼
┌──────────┐
│ 重排序    │ ← 对检索结果重新排序
└────┬─────┘
     │
     ▼
┌──────────┐
│ 上下文构建│ ← 组装Prompt + 检索结果
└────┬─────┘
     │
     ▼
┌──────────┐
│ LLM生成  │ ← 基于上下文生成回答
└────┬─────┘
     │
     ▼
┌──────────┐
│ 引用标注  │ ← 标注回答的出处
└──────────┘
     │
     ▼
  返回答案（含引用）
```

### 4.2 检索策略

```python
def retrieve_context(query, top_k=5):
    # 1. 向量检索（语义相似度）
    query_embedding = embed_model.encode(query)
    vector_results = vector_db.search(
        query_embedding, 
        top_k=top_k * 2
    )
    
    # 2. 关键词检索（BM25）
    keyword_results = bm25_search(query, top_k=top_k * 2)
    
    # 3. 混合检索（Hybrid Search）
    #    结合向量检索和关键词检索
    merged_results = merge_results(
        vector_results, 
        keyword_results,
        vector_weight=0.7,
        keyword_weight=0.3
    )
    
    # 4. 重排序（Reranker）
    #    使用Cross-Encoder对结果精排
    reranked = reranker.rerank(query, merged_results)
    
    # 5. 返回Top-K
    return reranked[:top_k]
```

## 六、技术实现

### 6.1 向量数据库选型

| 方案 | 优点 | 缺点 | 推荐场景 |
|------|------|------|---------|
| pgvector | 与PostgreSQL一体，运维简单 | 大规模性能一般 | MVP/小规模 |
| Milvus | 高性能，分布式 | 运维复杂 | 大规模 |
| Chroma | 轻量，Python原生 | 功能有限 | 开发调试 |

**推荐**：MVP阶段使用pgvector，后续根据数据量迁移Milvus

### 6.2 Embedding模型选型

| 模型 | 维度 | 中文效果 | 成本 |
|------|------|---------|------|
| text2vec-large-chinese | 1024 | 优秀 | 免费（本地） |
| 通义text-embedding-v2 | 1536 | 优秀 | 按量付费 |
| OpenAI text-embedding-3-small | 1536 | 良好 | 按量付费 |
| BGE-large-zh | 1024 | 优秀 | 免费（本地） |

**推荐**：优先使用BGE-large-zh或text2vec本地部署

### 6.3 对话管理

```python
class ConversationManager:
    def __init__(self, max_history=10):
        self.max_history = max_history
    
    def build_messages(self, history, new_question, context):
        messages = [
            {"role": "system", "content": SYSTEM_PROMPT}
        ]
        
        # 添加历史对话（最近N轮）
        for h in history[-self.max_history:]:
            messages.append({"role": "user", "content": h["question"]})
            messages.append({"role": "assistant", "content": h["answer"]})
        
        # 添加当前问题（含检索上下文）
        messages.append({
            "role": "user", 
            "content": USER_PROMPT.format(
                context=context,
                question=new_question
            )
        })
        
        return messages
```

## 七、异常处理

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 知识库为空 | 未上传文档 | 提示上传文档 |
| 检索无结果 | 问题超出知识库范围 | 告知无相关资料 |
| LLM超时 | 网络或服务异常 | 重试，提示稍后 |
| 幻觉回答 | LLM编造内容 | 要求引用原文 |
| 文档解析失败 | 格式不支持 | 提示支持格式 |
