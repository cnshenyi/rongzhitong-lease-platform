# 融智通 - 模块4：合同智能审核 详细设计

## 一、模块概述

### 1.1 业务背景
融资租赁合同是项目的法律基础，合同审核是风控的关键环节。

传统合同审核痛点：
- **人工审核慢**：法务逐条审核，耗时1-3天
- **遗漏风险高**：关键条款遗漏、表述不清
- **标准不统一**：不同法务审核标准不一
- **版本管理混乱**：合同修改多版本，难以追溯

### 1.2 核心功能
1. AI自动提取合同关键条款
2. 与标准模板对比，发现差异和风险点
3. 智能识别不利条款、缺失条款
4. 生成审核报告和修改建议
5. 支持合同版本对比

### 1.3 价值量化
| 指标 | 现状 | 使用后 | 提升 |
|------|------|--------|------|
| 合同审核时间 | 1-3天 | 10-30分钟 | 50倍 |
| 风险条款识别率 | 70% | 95%+ | +25% |
| 审核标准化程度 | 低 | 高 | 显著提升 |
| 法务工作量 | 100% | 20% | -80% |

## 三、合同类型与关键条款

### 3.1 融资租赁合同关键条款

| 条款类型 | 关键要素 | 风险点 |
|---------|---------|--------|
| **租赁物条款** | 设备名称、型号、数量、价值 | 描述不清、价值虚高 |
| **租金条款** | 租金总额、支付方式、支付时间 | 租金计算错误、支付条件不明 |
| **租赁期限** | 起租日、租赁期、到期处理 | 期限不合理、续租条件不清 |
| **所有权条款** | 租赁期间所有权归属 | 所有权约定不明 |
| **担保条款** | 担保方式、担保人、担保范围 | 担保不足、担保人资质差 |
| **违约责任** | 违约情形、违约金、救济措施 | 违约责任不对等、救济不足 |
| **保险条款** | 保险类型、投保人、受益人 | 保险缺失、受益人错误 |
| **维修保养** | 维修责任、费用承担 | 责任不清 |
| **提前终止** | 终止条件、结算方式 | 条件过于宽松 |
| **争议解决** | 管辖法院/仲裁机构 | 管辖不利 |

# 将合同条款与标准模板对比
def compare_with_template(contract_clauses, template_clauses):
    results = []
    
    for template_clause in template_clauses:
        # 在合同中查找对应条款
        matched = find_similar_clause(
            template_clause, 
            contract_clauses,
            similarity_threshold=0.7  # 相似度阈值
        )
        
        if matched:
            # 对比具体内容差异
            diff = compare_clause_content(
                template_clause, 
                matched
            )
            results.append({
                "clause_name": template_clause.name,
                "status": "存在",
                "difference": diff,
                "risk_level": assess_risk(diff)
            })
        else:
            # 缺失条款
            results.append({
                "clause_name": template_clause.name,
                "status": "缺失",
                "risk_level": "HIGH"
            })
    
    return results
```

**3.2 差异分析**
```
差异类型：
1. 缺失条款：模板有，合同没有
2. 多余条款：合同有，模板没有（可能是特殊约定）
3. 内容差异：条款存在但内容不同

## 六、版本对比功能

### 6.1 对比算法

```python
# 合同版本对比
def compare_contract_versions(version1, version2):
    # 1. 文本级对比（逐字对比）
    text_diff = difflib.unified_diff(
        version1.text.splitlines(),
        version2.text.splitlines()
    )
    
    # 2. 条款级对比（结构化对比）
    clause_diff = []
    for clause_name in version1.clauses.keys():
        v1_clause = version1.clauses[clause_name]
        v2_clause = version2.clauses.get(clause_name)
        
        if not v2_clause:
            clause_diff.append({
                "clause": clause_name,
                "change": "删除",
                "old": v1_clause,
                "new": None
            })
        elif v1_clause != v2_clause:
            clause_diff.append({
                "clause": clause_name,
                "change": "修改",
                "old": v1_clause,
                "new": v2_clause
            })
    
    # 3. 关键信息对比（金额、日期等）
    key_info_diff = compare_key_info(version1, version2)
    
    return {
        "text_diff": text_diff,
        "clause_diff": clause_diff,
        "key_info_diff": key_info_diff
    }
```
