# 融智通 - 模块1：租赁物清单识别 详细设计

## 一、模块概述

### 1.1 业务背景
融资租赁项目中，客户需提交租赁物（设备）清单作为项目申请材料。
这些清单格式五花八门（Excel、PDF表格、扫描件、手机拍照），
业务员需手动将清单信息录入系统，工作量大且容易出错。

### 1.2 核心功能
通过OCR+AI技术，自动识别各种格式的租赁物清单，
提取设备信息（名称、型号、数量、单价等），
生成标准化的结构化数据，支持人工校验和导出。

### 1.3 价值量化
| 指标 | 现状 | 使用后 | 提升 |
|------|------|--------|------|
| 单份清单录入时间 | 4-8小时 | 5-10分钟 | 48倍 |
| 数据准确率 | 95%（人工） | 97%+（AI+人工校验） | +2% |
| 单日处理量 | 1-2份 | 20+份 | 15倍 |
| 人力需求 | 1-2人 | 0.2人 | -80% |

### 3.2 详细步骤

#### 步骤1：上传文件
```
操作：
1. 用户进入"清单识别"页面
2. 点击上传区域 或 拖拽文件到上传区域
3. 选择一个或多个文件
4. 系统显示文件列表和上传进度
5. 上传完成后，自动开始识别处理（或用户点击"开始识别"）

校验规则：
- 文件格式：xlsx/xls/pdf/jpg/jpeg/png/docx
- 单个文件大小：≤100MB
- 单次上传数量：≤20个文件
- 同一文件不重复上传

错误处理：
- 格式不支持：提示"仅支持Excel/PDF/图片/Word格式"
- 文件过大：提示"文件大小不能超过100MB"
- 上传失败：提示"上传失败，请重试"
```

#### 步骤2：AI智能识别
```
处理过程（后端）：
1. 判断文件类型
   - Excel → 直接读取表格数据
   - PDF → 判断是文字型还是扫描型
     - 文字型 → 直接提取表格
     - 扫描型 → OCR识别
   - 图片 → OCR识别
   - Word → 提取表格

2. 表格结构识别
   - 识别表头行（字段映射）
   - 识别数据行（提取内容）
   - 识别合并单元格

3. LLM结构化处理
   - 将原始识别结果发送给LLM
   - LLM进行字段映射和规范化
   - 返回标准化JSON结构

4. 数据校验
   - 金额 = 数量 × 单价（自动校验）
   - 缺失字段标记
   - 异常值标记

页面展示：
- 显示识别进度（处理中/已完成）
- 预计耗时提示
- 处理完成后自动刷新结果
```

#### 步骤3：人工校验
```
操作：
1. 系统展示识别结果表格
2. 原文件在左侧预览，结果在右侧展示
3. 系统标注需要关注的行（低置信度、校验异常等）
4. 用户逐行检查，可直接在表格中修改
5. 修改后实时校验

特殊标注：
- 🔴 红色标记：校验错误（如金额不等于数量×单价）
- 🟡 黄色标记：低置信度（<80%），建议核对
- 🟢 绿色标记：校验通过
- 灰色标记：自动补全的字段
```

#### 步骤4：确认保存
```
操作：
1. 用户确认所有数据无误
2. 选择关联项目（可选）
3. 填写清单名称/备注（可选）
4. 点击"确认保存"
5. 数据保存到数据库
6. 生成操作日志

数据去重：
- 与已有清单对比
- 发现重复设备时提醒
```

#### 步骤5：导出使用
```
导出格式：
- 标准化Excel（.xlsx）
- PDF报告
- JSON数据（用于API对接）

Excel模板：
| 序号 | 设备名称 | 品牌 | 型号规格 | 单位 | 数量 | 单价(元) | 金额(元) | 生产厂家 | 备注 |
|------|---------|------|---------|------|------|---------|---------|---------|------|
| 1 | XXX设备 | XXX | XXX | 台 | 1 | 50,000 | 50,000 | XXX公司 | - |

自动汇总：
- 设备总数量
- 设备总金额
- 设备分类统计
```

## 五、AI识别策略

### 5.1 Excel文件处理

```python
# 处理流程
def process_excel(file_path):
    # 1. 读取所有sheet
    workbook = openpyxl.load_workbook(file_path)
    
    for sheet in workbook.sheetnames:
        ws = workbook[sheet]
        
        # 2. 识别表头行
        #    策略：遍历前10行，找到包含关键词最多的行
        #    关键词：设备名称/名称、型号/规格、数量、单价、金额...
        header_row = find_header_row(ws)
        
        # 3. 字段映射
        #    将各种可能的表头名映射到标准字段
        #    如："器械名称" → "name"，"规格型号" → "model"
        field_mapping = map_fields(ws[header_row])
        
        # 4. 提取数据行
        #    从表头下一行开始，逐行提取
        #    处理合并单元格、空行、汇总行
        data_rows = extract_data(ws, header_row, field_mapping)
        
        # 5. 数据清洗
        #    去除空白、统一格式、类型转换
        cleaned_data = clean_data(data_rows)
        
        # 6. 交给LLM做最终确认和补全
        result = llm_verify_and_complete(cleaned_data)
    
    return result
```

### 5.2 PDF/图片处理

```python
# 处理流程
def process_pdf_or_image(file_path):
    # 1. 判断PDF类型（文字型 vs 扫描型）
    if is_text_pdf(file_path):
        # 2a. 文字型PDF：直接提取表格
        tables = extract_tables_from_pdf(file_path)  # 使用pdfplumber
    else:
        # 2b. 扫描型PDF或图片：OCR识别
        images = convert_to_images(file_path)  # PDF转图片
        
        for image in images:
            # OCR识别
            ocr_result = paddleocr.ocr(image)
            # 表格结构识别
            tables = detect_table_structure(ocr_result)
    
    # 3. 将原始表格数据交给LLM进行结构化
    structured_data = llm_structure(tables)
    
    # 4. 数据校验和补全
    result = validate_and_complete(structured_data)
    
    return result
```

## 六、异常处理

### 6.1 识别异常

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 格式无法识别 | 文件损坏或格式不支持 | 提示用户重新上传 |
| 表格结构异常 | 多个表头、不规则合并 | 尝试多种识别策略，人工确认 |
| 字段缺失 | 必要字段未找到 | 标黄提醒，用户手动填写 |
| OCR识别错误 | 图片模糊、字迹不清 | 低置信度标注，用户核对 |
| 金额计算不符 | 金额≠数量×单价 | 标红提醒，用户修正 |

### 6.2 性能异常

| 异常类型 | 触发条件 | 处理方式 |
|---------|---------|---------|
| 文件过大 | >100MB | 拒绝上传，提示压缩 |
| 识别超时 | >60秒 | 转异步处理，完成后通知 |
| LLM调用失败 | API异常 | 重试3次，降级为规则引擎 |
| OCR服务不可用 | 服务宕机 | 切换备用OCR服务 |

```
┌──────┐  上传  ┌──────┐  识别  ┌──────┐  校验  ┌──────┐
│ 待上传 │──────>│ 识别中 │──────>│ 待校验 │──────>│ 已确认 │
└──────┘       └──┬───┘       └──┬───┘       └──────┘

                  ▼              ▼
               ┌──────┐      ┌──────┐
               │识别失败│      │重新识别│

### 状态说明
- **待上传**：初始状态
- **识别中**：AI正在处理
- **待校验**：识别完成，等待人工确认
- **已确认**：人工确认通过，数据已保存
- **识别失败**：识别过程出错
- **重新识别**：用户要求重新处理
